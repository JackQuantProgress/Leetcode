# NeetCode 150 – C++ (Quantitative Development Focus)

This repository contains solutions to the NeetCode 150 problem set implemented in C++, with emphasis on algorithmic rigor, performance awareness, and interview preparation for quantitative development and low-latency systems roles.

---

## Objective

Build strong technical foundations required for quantitative and systems engineering roles:

- Algorithmic problem solving under strict time constraints  
- Efficient implementation in modern C++ (C++17/20)  
- Performance intuition relevant to trading systems  
- Preparation for quant developer and HFT interviews  

---

## Language Choice: C++

C++ is used exclusively due to its relevance in production trading systems:

- High-frequency trading infrastructure  
- Low-latency market data systems  
- Real-time execution engines  
- Performance-critical backend systems  

Core focus areas:

- Memory control and deterministic performance  
- STL complexity awareness  
- Cache-conscious design patterns  
- Allocation-aware implementation choices  

---

## Engineering Principles

All solutions follow constraints similar to production-grade systems:

- Explicit time and space complexity control  
- Avoid unnecessary heap allocations  
- Prefer contiguous memory access patterns  
- Iterative solutions when recursion adds overhead risk  
- STL usage guided by complexity guarantees, not convenience  
- Awareness of constant-factor performance
